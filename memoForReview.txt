# 인터페이스
- 필드값은 기본 문법상 위로만(> 부모), 메서드는 오버라이딩 예외로 아래로 가능
- 따라서 참조 타입에 따라 필드값은 갈리지만 메서드는 오버라이딩 자식 우선(메서드에 따른 다이아몬드 문제가 구현엔 없음)

- 구현과 상속을 동시에 쓰면 extends .. implement ... 순서

- 순수추상클래스(자바엔 없는 개념) + 다중 구현 + 오버라이딩 제약 = 인터페이스
- 클래스, 추상클래스, 인터페이스 모두 코드/메모리 구조상 동일(기능 제약)
- 따라서 클래스처럼 매개변수로 인터페이스를 사용시 상속&구현된 자식 인스턴스도 사용 가능
